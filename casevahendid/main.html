Case vahendid

Mis on Case vahendid

CASE-vahendid (Computer-Aided Software Engineering) on tarkvaraarenduse tööriistad ja meetodid, mis toetavad süsteemide analüüsi, projekteerimist, arendust ja hooldust. Nende eesmärk on automatiseerida ja lihtsustada tarkvaraarenduse protsessi, parandades seeläbi töö efektiivsust ja kvaliteeti.

Mille jaoks neid kasutatakse

CASE-vahendeid kasutatakse tarkvara või koodi arenduse automatiseerimiseks ja lihtsustamiseks, muutes tarkvaraarenduse protsessi tõhusamaks ja sujuvamaks.

Kuidas jagatakse Case vahendid

CASE-vahendid jagatakse mitmeti:

  • Tarkvaraarenduse elutsükli järgi:
    • Ülemise taseme vahendid (Upper CASE): Nõuete kogumine ja haldamine, UML-diagrammid, süsteemidisain.
    • Alumise taseme vahendid (Lower CASE): Koodi genereerimine, testimine, veahaldus.
    • Terviklikud vahendid: Katavad kogu arendusprotsessi, nt Rational Rose.
  • Funktsionaalsuse järgi: Analüüs, projekteerimine, arendus, testimine, hooldus.
  • Kasutajaliidese järgi: Graafilised (nt diagrammide koostamine) ja tekstipõhised tööriistad.

Millised vahendid saab kasutada X arendusmudeli elutsüklis

Näiteks:

  • Kaskaadmudel: Visio, Rational Rose (disain); Selenium (testimine).
  • Agile-mudel: Jira (planeerimine); Visual Studio Code (arendus).
  • V-mudel: Lucidchart (disain); TestRail (testimine).

Milliseid vahendeid olen juba ise kasutanud

  • Visual Studio Code: Mitmekülgne IDE koodi kirjutamiseks ja arendamiseks.
  • GitHub: Koodireposid, koostöö ja CI/CD haldamine.
  • Python IDLE: Lihtne ja efektiivne Python-programmeerimise tööriist.
